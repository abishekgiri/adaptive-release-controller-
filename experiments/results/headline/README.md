# Headline-claim result files

These files are committed specifically so that the headline numerical claims in the top-level [`README.md`](../../../README.md) verification table can be checked on a fresh clone without re-running any experiment. Each file is a copy of a result produced by the experiment scripts; the complete set of per-seed result files lives under `experiments/results/<config>/` (blanket-gitignored) and is regenerated by running the corresponding script (`run_robustness.py`, `run_ablations.py`, `run_bandits.py`, `run_cost_sweep.py`). Filenames here encode the finding and seed(s); `real_github_actions/` holds the 30 per-seed summaries needed for the Thompson-vs-LinUCB comparison.
